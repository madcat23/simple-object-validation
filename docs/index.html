Examples

Simple registration form

import { assemble, chain, isRequired, isInteger, isGreaterThanOrEqual, isEmail } from 'simple-object-validation'

const isValidCustomer = assemble({
  name: isRequired('Name'),
  age: chain([
    isRequired,
    isInteger,
    isGreaterThanOrEqual(18)
  ])('Age'),
  email: chain([
    isRequired,
    isEmail,
  ])('Email'),
})

isValidCustomer({
  age: 17,
  email: 'foo'
})
// { name: 'Name is required.', age: 'Age must be greater than or equal 18.', email: 'Email must be a valid email address.' }

isValidCustomer({
  name: 'Mathew',
  age: 18,
  email: 'mathew@email.com'
})
// {}

Core API

validator

(checkFunction: any => boolean, messageCreator: (param: any, name: string, value: any) => string)
=> (param: any) => (name: string) => (value: any) => string | undefined

Okay, okay! This looks difficult at first sight, definitely. But it totally makes sense. This is how it works:

In simple-object-validation every actual validation function takes a value as an argument and returns either undefined or an error message. It really is that simple. But in order to get to this simple function you will need to configure your wanted validation behaviour. This is done via higher order functions. These higher order functions are functions that return other functions. So for example:

import { validator } from 'simple-object-validation'

// Example:
const checkFunction = value => value === 42
const messageCreator = (param, name, value) => `${name} must be the meaning of life!`
const isMeaningOfLife = validator(checkFunction, messageCreator)

isMeaningOfLife('Magic Number')(3.14) // 'Magic number must be the meaning of life'
isMeaningOfLife('Magic Number')(42) // undefined
